# Memory

This directory contains persistent memory and learnings from the project.

## Purpose

Store `.memory.md` files that capture important learnings, patterns, and insights that should be remembered across development sessions.

## Structure

Memory files can be organized by:
- Topic or domain
- Time period
- Component or feature
- Type of learning

## Memory File Format

Memory files should follow this format:

```markdown
# Memory: [Title]

## Learnings
Important lessons learned during development.

## Decisions
Key decisions made and their rationale.

## Patterns
Recurring patterns or best practices discovered.
## Gotchas
Common pitfalls or issues to avoid.

## Resources
Useful resources and references.

## Timestamp
When this memory was created or last updated.
```

## Types of Memory

### Technical Memory
- Code patterns that work well
- Performance considerations
- Integration details
- Tool usage tips

### Process Memory
- Workflow improvements
- Communication patterns
- Collaboration insights
- Development practices

### Domain Memory
- Business logic insights
- User requirements
- Domain-specific knowledge
- Terminology and concepts

## Best Practices

1. **Record Promptly**: Capture learnings while they're fresh
2. **Be Specific**: Include concrete examples and details
3. **Add Context**: Explain why something is important
4. **Update Regularly**: Revise as understanding evolves
5. **Cross-Reference**: Link to related memories and documentation

## Using Memory Files

Memory files help:
- Avoid repeating mistakes
- Share knowledge across the team
- Onboard new contributors faster
- Make better decisions
- Build on past learnings
- Provide context to AI assistants
