# Context

This directory contains context-specific documentation for different areas of the project.

## Purpose

Store `.context.md` files and other context-related documentation that helps maintain understanding of different project areas.

## Structure

Context files can be organized by:
- Feature area
- Component
- Subsystem
- Development phase

## Context File Format

Context files should follow this format:

```markdown
# Context: [Title]

## Current State
Description of the current state of this area.

## Relevant Information
Important information relevant to this context.

## Related Files
- List of related files
- Links to relevant code
- References to specifications

## Recent Changes
Brief history of recent changes (optional).

## Next Steps
Planned work or considerations (optional).
```

## Best Practices

1. **Keep Current**: Update context files as the project evolves
2. **Be Specific**: Focus on relevant information for the specific context
3. **Link Resources**: Reference related files and documentation
4. **Clean Up**: Remove outdated context periodically
5. **Version Control**: Track all changes to context files

## Using Context Files

Context files help:
- Onboard new contributors
- Remember project decisions
- Maintain understanding across sessions
- Provide background for AI assistants
- Document tribal knowledge
